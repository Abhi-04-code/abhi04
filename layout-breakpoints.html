BOOTSTRAP LAYOUT BREAKPOINTS

▶ Introduction to Breakpoints

Breakpoints are the building blocks of responsive design in Bootstrap. They are predefined screen sizes at which your layout and components adapt to provide the best possible user experience across different devices.

Bootstrap uses a mobile-first approach, meaning styles are applied for mobile devices first, then enhanced for larger screens using media queries.

Pro Tip: Bootstrap 5 includes 6 breakpoints: X-Small, Small, Medium, Large, Extra Large, and Extra Extra Large. Each breakpoint corresponds to common device screen sizes.

▶ Using Breakpoints in CSS

Media Query Syntax

Bootstrap uses Sass mixins to generate media queries. Here's how they work:

// Using Sass mixins
.element {
  // Mobile-first styles (applies to all screens)
  padding: 1rem;
  
  // Medium screens and up
  @include media-breakpoint-up(md) {
    padding: 2rem;
  }
  
  // Large screens and up
  @include media-breakpoint-up(lg) {
    padding: 3rem;
  }
}

// Generated CSS
.element {
  padding: 1rem;
}

@media (min-width: 768px) {
  .element {
    padding: 2rem;
  }
}

@media (min-width: 992px) {
  .element {
    padding: 3rem;
  }
}

Breakpoint Directions

Bootstrap provides mixins for different breakpoint directions:

// Breakpoint UP (min-width)
@include media-breakpoint-up(sm) { ... }
// Applies to screens ≥576px

// Breakpoint DOWN (max-width)
@include media-breakpoint-down(md) { ... }
// Applies to screens <768px

// Breakpoint ONLY (min-width and max-width)
@include media-breakpoint-only(lg) { ... }
// Applies to screens between 992px and 1199px

// Breakpoint BETWEEN (min-width and max-width)
@include media-breakpoint-between(md, xl) { ... }
// Applies to screens between 768px and 1199px

▶ Breakpoints in Utility Classes

Responsive Utilities

Most Bootstrap utility classes can be made responsive by adding breakpoint infixes:

<!-- No breakpoint (applies to all screens) -->
<div class="text-center">Always centered</div>

<!-- Small breakpoint and up -->
<div class="text-sm-start">Left aligned on sm+</div>

<!-- Medium breakpoint and up -->
<div class="text-md-center">Centered on md+</div>

<!-- Large breakpoint and up -->
<div class="text-lg-end">Right aligned on lg+</div>

<!-- Extra large breakpoint and up -->
<div class="text-xl-center">Centered on xl+</div>

▶ Customizing Breakpoints

Sass Variables

You can customize breakpoints by modifying Sass variables:

// _variables.scss

// Custom breakpoint values
$grid-breakpoints: (
  xs: 0,
  sm: 576px,
  md: 768px,
  lg: 992px,
  xl: 1200px,
  xxl: 1400px
);

// Custom container maximum widths
$container-max-widths: (
  sm: 540px,
  md: 720px,
  lg: 960px,
  xl: 1140px,
  xxl: 1320px
);

// Import Bootstrap after customizing variables
@import "bootstrap/scss/bootstrap";

Adding Custom Breakpoints

Extend the breakpoint map to include custom breakpoints:

// Add custom breakpoint
$grid-breakpoints: map-merge(
  (
    xxxl: 1600px
  ),
  $grid-breakpoints
);

// Add corresponding container max-width
$container-max-widths: map-merge(
  (
    xxxl: 1540px
  ),
  $container-max-widths
);

// Now you can use the custom breakpoint
.custom-element {
  @include media-breakpoint-up(xxxl) {
    font-size: 1.5rem;
  }
}

Navigation Example

Responsive navigation that changes behavior at different breakpoints:

  <div class="container">
    <a class="navbar-brand" href="#">My App</a>
    
    <!-- Collapses on screens smaller than lg -->
    <button class="navbar-toggler" type="button" data-bs-toggle="collapse" data-bs-target="#navbarNav">
      <span class="navbar-toggler-icon"></span>
    </button>
    
    <div class="collapse navbar-collapse" id="navbarNav">
      <ul class="navbar-nav">
        <li class="nav-item">

        </li>

          <a class="nav-link" href="#">About</a>
        </li>
      </ul>

  </div>
</nav>

Breakpoint Best Practices

  • Mobile-First: Always design for mobile first, then enhance for larger screens.
  • Consistent Grid: Use the grid system with breakpoints for consistent layouts.
  • Test Thoroughly: Test your responsive design on actual devices when possible.
  • Semantic Breakpoints: Choose breakpoints based on content, not specific devices.
