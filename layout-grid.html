BOOTSTRAP LAYOUT GRID SYSTEM

▶ Introduction to the Grid System

Bootstrap's grid system is built with Flexbox and is fully responsive. It uses a series of containers, rows, and columns to layout and align content. The grid system is mobile-first and scales up to 12 columns as the device or viewport size increases.

The grid system is the foundation for creating complex, responsive layouts without writing custom CSS. It's designed to work seamlessly across all screen sizes and devices.

Pro Tip: Bootstrap's grid uses a 12-column system that can be combined and nested to create virtually any layout you can imagine. Remember the hierarchy: Container → Row → Column.

▶ Grid System Components

Containers

Provide a means to center and horizontally pad your site's contents.

.container .container-fluid
Rows

Wrappers for columns that ensure proper alignment and spacing.

.row
Columns

The building blocks of the grid system that hold your actual content.

▶ Responsive Grid System

Breakpoint-based Grid Classes

Bootstrap's grid includes five tiers of predefined classes for building complex responsive layouts:

Breakpoint Class Infix Dimensions Example
X-Small None <576px .col-*
Small sm ≥576px .col-sm-*
Medium md ≥768px .col-md-*
Large lg ≥992px .col-lg-*
X-Large xl ≥1200px .col-xl-*
XX-Large xxl ≥1400px .col-xxl-*

Complex Responsive Example

Create layouts that change at different breakpoints:

<div class="container">
  <div class="row">
    <!-- Stack on mobile, 50% on tablet, 33.3% on desktop -->
    <div class="col-12 col-md-6 col-lg-4">
      <div class="col-demo">Column 1</div>
    </div>
    <div class="col-12 col-md-6 col-lg-4">
      <div class="col-demo">Column 2</div>
    </div>
    <div class="col-12 col-md-12 col-lg-4">
      <div class="col-demo">Column 3</div>

Grid System Best Practices

  • Mobile-First: Design for mobile first, then use breakpoint classes for larger screens.
  • Proper Hierarchy: Always use Container → Row → Column structure.
  • Column Math: Ensure columns in each row add up to 12 or fewer.
  • Responsive Thinking: Consider how your layout should adapt at each breakpoint.
