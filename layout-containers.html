BOOTSTRAP LAYOUT CONTAINERS

▶ Introduction to Containers

Containers are the most basic layout element in Bootstrap and are required when using the default grid system. They contain, pad, and align your content within a given device or viewport.

Bootstrap provides three types of containers that respond to different breakpoints and use cases. Containers are essential for creating consistent, responsive layouts.

Pro Tip: Containers are the foundation of Bootstrap's grid system. They provide horizontal padding and center your content, ensuring proper alignment across different screen sizes.

▶ Container Types

Default Container

Responsive fixed-width container that changes max-width at each breakpoint.

.container
Fluid Container

Full-width container that spans the entire width of the viewport.

.container-fluid
Responsive Containers

Containers that are 100% wide until specified breakpoints.

.container-{breakpoint}
Container Type Class Width Use Case
Default .container Responsive fixed width Most common use case
Fluid .container-fluid 100% width always Full-width layouts
Responsive .container-{breakpoint} 100% wide until breakpoint Custom responsive behavior

▶ Default Container Behavior

Default Container Max-Widths

The default .container class has responsive max-widths that change at each breakpoint:

// Default container max-widths in Sass
$container-max-widths: (
  sm: 540px,
  md: 720px,
  lg: 960px,
  xl: 1140px,
  xxl: 1320px
);
Breakpoint Max Width Description
X-Small (<576px) 100% (no max-width) Full width on mobile
Small (≥576px) 540px Fixed width on small screens
Medium (≥768px) 720px Fixed width on tablets
Large (≥992px) 960px Fixed width on desktops
X-Large (≥1200px) 1140px Fixed width on large desktops
XX-Large (≥1400px) 1320px Fixed width on extra large displays

Default Container Example

Here's how the default container behaves across different screen sizes:

<div class="container">
  <!-- Your content here -->
  <h1>My Page Title</h1>
  <p>This content is contained within a responsive container.</p>
</div>

▶ Practical Container Examples

Basic Page Layout

Standard page structure using containers:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website</title>
  <link href="bootstrap.css" rel="stylesheet">
</head>
<body>
  <!-- Navigation -->
  <nav class="navbar navbar-expand-lg navbar-dark bg-dark">
    <div class="container">
      <a class="navbar-brand" href="#">My Logo</a>
      <!-- Nav content -->
    </div>
  </nav>

  <!-- Main Content -->
  <main class="container my-5">
    <div class="row">
      <div class="col-md-8">
        <h1>Welcome to My Site</h1>
        <p>Main content area...</p>
      </div>
      <div class="col-md-4">
        <div class="card">
          <div class="card-body">
            <h5 class="card-title">Sidebar</h5>
            <p class="card-text">Sidebar content...</p>
          </div>
        </div>
      </div>
    </div>
  </main>

  <!-- Footer -->
  <footer class="bg-light text-dark py-4">
    <div class="container">
      <div class="row">
        <div class="col-md-6">
          <p>© 2024 My Website</p>
        </div>
        <div class="col-md-6 text-md-end">
          <a href="#" class="text-dark">Privacy Policy</a>
        </div>
      </div>
    </div>
  </footer>
</body>
</html>

▶ Customizing Containers

Custom Container Sizes

Modify container max-widths using Sass variables:

// _variables.scss

// Custom container max-widths
$container-max-widths: (
  sm: 540px,
  md: 720px,
  lg: 980px,    // Changed from 960px
  xl: 1160px,   // Changed from 1140px
  xxl: 1340px   // Changed from 1320px
);

// Custom container padding
$container-padding-x: 2rem; // Default is 1.5rem (12px)

// Import Bootstrap
@import "bootstrap/scss/bootstrap";

Creating Custom Container Classes

Add your own container variants:

// _custom-containers.scss

// Extra small container
.container-xs {
  max-width: 440px;
  margin: 0 auto;
  padding: 0 $container-padding-x;
}

// Extra large container  
.container-xxxl {
  max-width: 1600px;
  margin: 0 auto;
  padding: 0 $container-padding-x;
}

// Usage in HTML
<div class="container-xs">
  <!-- Narrow content -->
</div>

<div class="container-xxxl">
  <!-- Extra wide content -->
</div>

Container Best Practices

  • Always Use Containers: Wrap your content in containers for proper padding and alignment.
  • Mobile-First: Default container works great for mobile-first responsive design.
  • Choose Wisely: Use fluid containers for full-width sections and default for contained content.
  • Grid Inside: Place grid rows and columns inside containers, not the other way around.
