BOOTSTRAP CLOSE BUTTON

▶ Close Button Overview

Bootstrap close buttons are generic close icons for dismissing content like modals, alerts, and toasts. They're built with minimal markup and styled to be easily customizable for different contexts and use cases.

Pro Tip: Close buttons are essential for user experience, providing clear ways to dismiss temporary content without disrupting the main workflow.

<script>
function dismissElement(button) {
  const element = button.closest('.dismissible-element');
  element.classList.add('fade-out');
  setTimeout(() => element.remove(), 300);
}

function dismissBadge(button) {
  const badge = button.closest('.badge');
  badge.style.opacity = '0';
  badge.style.transform = 'scale(0.8)';
  setTimeout(() => badge.remove(), 300);
}

function addNewElement() {
  const container = document.getElementById('dynamicElements');
  const newElement = document.createElement('div');
  newElement.className = 'dismissible-element';
  newElement.innerHTML = `
    <div class="d-flex justify-content-between align-items-center">
      <span>New element added at ${new Date().toLocaleTimeString()}</span>
      <button type="button" class="btn-close" onclick="dismissElement(this)"></button>
    </div>
  `;
  container.appendChild(newElement);
}
</script>

▶ Close Button Options

Close Button Classes and Attributes

Class/Attribute Description Usage
.btn-close Base close button class Required for all close buttons
.btn-close-white White variant for dark backgrounds Use on dark-colored components
disabled Disabled state Makes close button non-interactive
aria-label="Close" Accessibility label Required for screen readers
data-bs-dismiss Auto-dismiss functionality alert, modal, toast, etc.
.btn-close-sm / .btn-close-lg Size variants Small and large close buttons

Close Button Best Practices

  • Clear Purpose: Always use close buttons for dismissible content only.
  • Accessibility: Include proper aria-label attributes for screen readers.
  • Consistent Placement: Place close buttons in predictable locations (top-right corner).
  • Visual Feedback: Provide hover and focus states for better user interaction.
