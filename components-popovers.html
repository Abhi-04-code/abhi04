BOOTSTRAP POPOVERS

▶ Popovers Overview

Bootstrap popovers are small overlays of content for housing secondary information. They're similar to tooltips but can contain more content including titles, and are triggered by clicking rather than hovering.

Pro Tip: Popovers rely on the third-party library Popper for positioning. You must initialize popovers yourself, and they require the data-bs-toggle="popover" attribute.

▶ Basic Popover

Simple Popover Example

<!-- Basic popover -->
<button type="button" class="btn btn-primary" 
        data-bs-toggle="popover" 
        data-bs-title="Popover Title" 
        data-bs-content="This is the basic popover content.">
    Click to toggle popover
</button>

<script>
// Initialize popovers
var popoverTriggerList = [].slice.call(document.querySelectorAll('[data-bs-toggle="popover"]'))
var popoverList = popoverTriggerList.map(function (popoverTriggerEl) {
    return new bootstrap.Popover(popoverTriggerEl)
})
</script>

▶ Custom Content

Advanced Popover with Custom Elements

<!-- Custom content popover -->
<button type="button" class="btn btn-warning" id="customPopover">
    Custom Content Popover
</button>

<template id="popover-template">
    <div class="popover-content">
        <div class="d-flex align-items-center mb-2">
            <i class="fas fa-exclamation-triangle text-warning me-2"></i>
            <strong>Custom Popover</strong>
        </div>
        <p class="mb-2">This popover has custom structured content.</p>
        <div class="alert alert-sm alert-info mb-2">
            <small>This is an alert inside the popover!</small>
        </div>
        <div class="d-grid gap-1">
            <button class="btn btn-sm btn-primary">Action 1</button>
            <button class="btn btn-sm btn-outline-secondary">Action 2</button>
        </div>
    </div>
</template>

<script>
document.addEventListener('DOMContentLoaded', function() {
    const customPopover = document.getElementById('customPopover');
    const template = document.getElementById('popover-template');
    
    const popover = new bootstrap.Popover(customPopover, {
        html: true,
        title: 'Custom Popover Title',
        content: template.innerHTML
    });
});
</script>

▶ JavaScript Methods

Programmatic Control

<script>
// Get popover instance
const popoverElement = document.getElementById('popoverButton');
const popover = new bootstrap.Popover(popoverElement, {
    title: 'Programmatic Popover',
    content: 'This popover is controlled via JavaScript methods.'
});

// Show popover
function showPopover() {
    popover.show();
}

// Hide popover
function hidePopover() {
    popover.hide();
}

// Toggle popover
function togglePopover() {
    popover.toggle();
}

// Update content
function updatePopover() {
    popover.setContent({
        '.popover-header': 'Updated Title',
        '.popover-body': 'This content has been updated programmatically!'
    });
}

// Disable popover
function disablePopover() {
    popover.disable();
}

// Enable popover
function enablePopover() {
    popover.enable();
}

// Event listeners
popoverElement.addEventListener('show.bs.popover', () => {
    console.log('Popover is about to be shown');
});

popoverElement.addEventListener('shown.bs.popover', () => {
    console.log('Popover is now fully shown');
});

popoverElement.addEventListener('hide.bs.popover', () => {
    console.log('Popover is about to be hidden');
});

popoverElement.addEventListener('hidden.bs.popover', () => {
    console.log('Popover is now fully hidden');
});
</script>

Popovers Best Practices

  • Accessibility: Always include proper ARIA labels and ensure keyboard navigation works.
  • User Experience: Use popovers for supplementary information, not critical content.
  • Content Strategy: Keep popover content concise and relevant to the trigger element.
  • Performance: Initialize popovers only when needed and dispose of them when not in use.
