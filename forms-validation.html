BOOTSTRAP FORM VALIDATION

▶ Form Validation Overview

Bootstrap provides powerful form validation styles and feedback messages using CSS and HTML5 validation features. You can implement both client-side and server-side validation with consistent styling.

Pro Tip: Bootstrap's validation styles work with HTML5 form validation and can be triggered with the .was-validated class or through JavaScript.

<script>
// Example starter JavaScript for disabling form submissions if there are invalid fields
(() => {
  'use strict'
  const forms = document.querySelectorAll('.needs-validation')
  Array.from(forms).forEach(form => {
    form.addEventListener('submit', event => {
      if (!form.checkValidity()) {
        event.preventDefault()
        event.stopPropagation()
      }
      form.classList.add('was-validated')
    }, false)
  })
})()
</script>

▶ Validation Types

Supported Validation Types

Validation Type HTML Attribute Description
Required required Field must be filled out
Email type="email" Must be a valid email address
URL type="url" Must be a valid URL
Pattern pattern="[A-Za-z]{3}" Must match specified regex pattern
Min/Max Length minlength="3", maxlength="10" Text length constraints
Min/Max Value min="1", max="100" Number range constraints

Form Validation Best Practices

  • Client & Server Validation: Always implement both client-side and server-side validation for security.
  • Clear Messages: Provide specific, helpful error messages that guide users to fix issues.
  • Visual Feedback: Use both color and icons to indicate validation states clearly.
  • Timely Validation: Validate on blur or as users type for immediate feedback.
