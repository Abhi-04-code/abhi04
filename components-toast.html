BOOTSTRAP TOAST

▶ Toast Overview

Bootstrap Toasts are lightweight notifications designed to mimic push notifications from mobile and desktop operating systems. They're flexible, responsive, and can be positioned anywhere on the page.

Pro Tip: Toasts are hidden by default. Use JavaScript to trigger them when needed, and they'll automatically dismiss after a timeout or when manually closed.

<!-- Trigger Button -->
<button type="button" class="btn btn-primary" id="liveToastBtn">Show toast</button>

<script>
const toastTrigger = document.getElementById('liveToastBtn')
const toastLiveExample = document.getElementById('liveToast')

if (toastTrigger) {
  toastTrigger.addEventListener('click', () => {
    const toast = new bootstrap.Toast(toastLiveExample)
    toast.show()
  })
}
</script>

<script>
// JavaScript to handle stacking
function addToastToStack() {
  const stack = document.getElementById('toastStack');
  const newToast = document.createElement('div');
  newToast.className = 'toast show mb-3';
  newToast.innerHTML = `
    <div class="toast-header">
      <strong class="me-auto">New Message</strong>
      <button type="button" class="btn-close" data-bs-dismiss="toast"></button>
    </div>
    <div class="toast-body">
      New toast added to stack!
    </div>
  `;
  stack.appendChild(newToast);
}
</script>

Toast Best Practices

  • Appropriate Use: Use toasts for non-critical notifications that don't require immediate action.
  • Timing: Set appropriate auto-hide delays (3-5 seconds for simple notifications).
  • Positioning: Choose positions that don't interfere with user interactions.
  • Consistent Styling: Use consistent colors and icons for different notification types.
  • Stacking: Implement proper stacking behavior for multiple simultaneous toasts.
