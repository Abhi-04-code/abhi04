BOOTSTRAP MODALS

▶ Modal Overview

Bootstrap modals are flexible dialog prompts that appear over the page content. They can contain any HTML content and are typically used for notifications, forms, or additional information that requires user interaction.

Pro Tip: Modals are built with HTML, CSS, and JavaScript and are positioned over everything else in the document.

▶ Basic Modal

Simple Modal Example

Modal title

Modal body text goes here.

<!-- Button trigger modal -->
<button type="button" class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#exampleModal">
  Launch demo modal
</button>

<!-- Modal -->
<div class="modal fade" id="exampleModal" tabindex="-1" aria-labelledby="exampleModalLabel" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-header">
        <h5 class="modal-title" id="exampleModalLabel">Modal title</h5>
        <button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>
      </div>
      <div class="modal-body">
        <p>Modal body text goes here.</p>
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Close</button>
        <button type="button" class="btn btn-primary">Save changes</button>
      </div>
    </div>
  </div>
</div>

▶ Modal JavaScript Methods

Programmatic Control

// JavaScript methods for modal control

// Get the modal instance
const myModal = new bootstrap.Modal(document.getElementById('myModal'));

// Show the modal
myModal.show();

// Hide the modal
myModal.hide();

// Toggle the modal
myModal.toggle();

// Event listeners
const myModalEl = document.getElementById('myModal');
myModalEl.addEventListener('show.bs.modal', event => {
  // Do something when modal is about to show
});

myModalEl.addEventListener('shown.bs.modal', event => {
  // Do something when modal is fully shown
});

myModalEl.addEventListener('hide.bs.modal', event => {
  // Do something when modal is about to hide
});

myModalEl.addEventListener('hidden.bs.modal', event => {
  // Do something when modal is fully hidden
});

Modal Best Practices

  • Accessibility: Always include proper ARIA attributes and ensure keyboard navigation works.
  • User Experience: Use modals sparingly and only for important interactions.
  • Visual Design: Ensure modal content is clear and actions are obvious.
  • Performance: Load modal content dynamically if it contains heavy resources.
