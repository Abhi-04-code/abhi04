BOOTSTRAP INPUT GROUPS

▶ Input Groups

Input groups allow you to add text, buttons, or button groups on either side of textual inputs, custom file inputs, or custom selects. They're perfect for enhancing form inputs with additional context or functionality using the .input-group class.

Pro Tip: Use .input-group with .input-group-text for text add-ons or .btn for button add-ons to create cohesive input groups.

▶ Basic Input Groups

Text Add-ons

https://example.com/users/
@example.com 
<!-- Prefix text -->
<div class="input-group">
  <span class="input-group-text">https://example.com/users/</span>
  <input type="text" class="form-control" id="basic-url">
</div>

<!-- Suffix text -->
<div class="input-group">
  <input type="email" class="form-control" placeholder="username">
  <span class="input-group-text">@example.com</span>
</div>

▶ Button Add-ons

Buttons in Input Groups

  • Action
  • Another action
  • Something else here
  • Separated link
<!-- Button prefix -->
<div class="input-group">
  <button class="btn btn-outline-secondary" type="button">Button</button>
  <input type="text" class="form-control">
</div>

<!-- Dropdown suffix -->
<div class="input-group">
  <input type="text" class="form-control">
  <button class="btn btn-outline-secondary dropdown-toggle" type="button" data-bs-toggle="dropdown">Dropdown</button>
  <ul class="dropdown-menu">
    <li><a class="dropdown-item" href="#">Action</a></li>
  </ul>
</div>

▶ Multiple Add-ons

Multiple Inputs and Add-ons

$ 0.00
First and last name 
<!-- Multiple text add-ons -->
<div class="input-group">
  <span class="input-group-text">$</span>
  <span class="input-group-text">0.00</span>
  <input type="text" class="form-control">
</div>

<!-- Multiple inputs -->
<div class="input-group">
  <span class="input-group-text">First and last name</span>
  <input type="text" class="form-control" aria-label="First name">
  <input type="text" class="form-control" aria-label="Last name">
</div>

▶ Checkboxes and Radios

Checkbox and Radio Add-ons

<!-- Checkbox add-on -->
<div class="input-group">
  <div class="input-group-text">
    <input class="form-check-input mt-0" type="checkbox">
  </div>
  <input type="text" class="form-control">
</div>

<!-- Radio add-on -->
<div class="input-group">
  <div class="input-group-text">
    <input class="form-check-input mt-0" type="radio">
  </div>
  <input type="text" class="form-control">
</div>

▶ File Input

File Input with Add-ons

<!-- Default file input -->
<div class="input-group">
  <input class="form-control" type="file" id="formFile">
</div>

<!-- Multiple files -->
<div class="input-group">
  <input class="form-control" type="file" id="formFileMultiple" multiple>
</div>

<!-- Disabled file input -->
<div class="input-group">
  <input class="form-control" type="file" id="formFileDisabled" disabled>
</div>

Input Group Best Practices

  • Clear Context: Use add-ons to provide context about the expected input format.
  • Consistent Sizing: Ensure input groups match the size of other form elements on the page.
  • Accessibility: Use proper aria-labels for screen readers when add-ons don't provide sufficient context.
  • Logical Grouping: Group related inputs together for better user experience.
