BOOTSTRAP BUTTONS

▶ Button Overview

Bootstrap buttons are versatile components used to trigger actions throughout your application. They come in various styles, sizes, and states to fit different use cases and design requirements.

Pro Tip: Use semantic button colors to convey meaning - primary for main actions, success for positive actions, danger for destructive actions, etc.

▶ Button Options

Button Classes and Attributes

Class/Attribute Description Usage
.btn Base button class Required for all buttons
.btn-[color] Button color variant primary, success, danger, warning, etc.
.btn-outline-[color] Outline button variant Transparent background with colored border
.btn-lg / .btn-sm Button size modifiers Large and small button sizes
disabled Disabled state attribute Makes button non-interactive
data-bs-toggle="button" Toggle button functionality Creates toggleable button state

Button Best Practices

  • Clear Action Labels: Use descriptive text that clearly indicates what the button does.
  • Consistent Sizing: Maintain consistent button sizes throughout your application.
  • Visual Hierarchy: Use primary colors for main actions and secondary colors for less important actions.
  • Loading States: Provide visual feedback during asynchronous operations.
